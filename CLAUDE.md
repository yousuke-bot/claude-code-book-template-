# CLAUDE.md

このファイルはClaude Codeがこのリポジトリで作業する際の指示書です。

## このプロジェクトについて

クライアントからWebサイト制作を受注し、納品するためのリポジトリです。
制作・保守管理・SEO対策まで一貫して対応します。

## クライアント案件の進め方

### 1. ヒアリング（聞くこと）
新しい案件を始めるときは必ず以下を確認する：

- 業種・会社名
- サイトの目的（集客・採用・EC・情報発信など）
- ターゲット（年齢層・地域・性別）
- 希望ページ数・ページ内容
- 参考にしたいサイトのURL
- 納期・予算
- 更新頻度（自分で更新するか、こちらで更新するか）

### 2. フォルダ作成
案件ごとに `clients/` 以下にフォルダを作る：

```
clients/
└── クライアント名/
    ├── index.html      # トップページ
    ├── about.html      # 会社概要
    ├── service.html    # サービス
    ├── contact.html    # お問い合わせ
    └── assets/
        ├── style.css
        ├── main.js
        └── images/
```

### 3. 制作ルール

**HTML/CSS**
- 外部ライブラリは使わない（依存なしのシンプル構成）
- スマートフォン対応（レスポンシブ）を必ず入れる
- フォントは `'Hiragino Sans', 'Yu Gothic', sans-serif` を使う
- CSSはカラー変数（`--primary`, `--accent` など）で管理する

**デザイン**
- シンプル・モダンを基本とする
- クライアントの業種に合わせたカラーを使う（例：士業＝紺、飲食＝暖色）
- 文字サイズは本文16px以上、行間1.7以上

**SEO**
- `<title>` と `<meta name="description">` を必ず設定する
- 見出しは `h1` → `h2` → `h3` の順番を守る
- 画像には `alt` 属性を必ず入れる

### 4. 納品前チェック
- [ ] スマートフォンで表示確認
- [ ] 問い合わせフォームの動作確認
- [ ] リンク切れがないか確認
- [ ] ページ読み込み速度の確認
- [ ] SEO設定（title・description）の確認

## ディレクトリ構成

```
/
├── company/        # 自社紹介ページ
│   └── index.html
├── clients/        # クライアント納品サイト
│   └── クライアント名/
├── ai-secretary/   # AI秘書ツール（開発中）
└── CLAUDE.md       # この指示書
```

## 開発環境の起動

```bash
npx http-server . -p 8080
```

ブラウザで http://localhost:8080 を開く。
